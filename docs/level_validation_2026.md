# 混雑レベルの検証記録(2019年基準 × 2024〜26年統計)

- 検証日: 2026-07-17
- 目的: 混雑レベルの基準である「全国の人流オープンデータ(国交省・2019年)」が、
  2026年現在も各スポットの**相対的な混雑序列**として妥当かを、最新の公表統計・報道と
  突き合わせて検証し、ズレが確認できたスポットを出典つきで補正する

## なぜ2019年データを使い続けるのか(前提)

1. **より新しい同等データが存在しない**。同データセットはコロナ対策調査の一環で
   公開されたもので、対象は2019〜2021年のみ。**2022年以降の公開予定はないと明言**されている
   ([G空間情報センター](https://www.geospatial.jp/ckan/dataset/mlit-1km-fromto))。
   2020・21年は行動制限で観光行動が極端に歪んでいるため、2019年が最も平常に近い基準となる
2. 代替候補の調査結果(2026-07):
   - 擬似人流データ(東大CSIS・2024年公開) — 国勢調査ベースの**合成データ**で、
     訪日観光客の行動が事実上含まれない。共同研究申請制で公開アプリへの利用可否も不明確 → 不採用
   - モバイル空間統計・Agoop等の商用人流データ — 有償・再配布不可 → 応募作品に使えない
   - ODPTの乗降者数(PassengerSurvey) — 年次合計のみで時間帯・平休日の区別がなく、
     駅利用者は観光滞在の代理として不適 → 検証の補助のみ
3. 本アプリが2019年データから使うのは**絶対量ではなく相対値**(スポット間の混雑比)。
   行程の並べ替えに効くのはこの相対値だけで、観光地間の序列は構造的に安定している
4. 2025年の訪日外国人は4,268万人と**2019年(3,188万人)比+34%の過去最高**
   ([JNTO発表・観光庁](https://www.mlit.go.jp/kankocho/tokei_hakusyo/shutsunyukokushasu.html))。
   主要スポットは当時より混んでいる方向のズレであり、混雑回避アプリとしては安全側

## 検証方法

- 各スポットについて「2019年以降に混雑構造を変える出来事(新施設の開業・閉鎖・大規模改変)が
  あったか」を公表資料・報道で確認
- 構造変化が**なければ相対値は据え置き**(訪日全体の増加は全スポットに掛かるため序列を変えない)
- 構造変化が**あれば**、年間来場者数など公表数値をもとに、既存スポットとの相対比較で
  保守的に補正(補正値は実測ではなく推定。crowd_profiles.jsonのlevel_noteにも明記)

## 補正値の決め方(2件)

- **豊洲市場 36/33 → 52/56**: 千客万来の購入者数(366万人/2025年)は、
  谷中銀座や柴又を大きく超え、お台場(47/50)以上・築地場外(83/56)未満の集客水準。
  観光型施設のため休日>平日とした
- **チームラボプラネッツ 56/49 → 62/64**: 年間250万人はスカイツリー(67/71)に迫る水準。
  お台場(47/50)を明確に上回り、スカイツリーは超えない帯に置いた。休日>平日

## 残る限界(READMEにも記載)

- 補正値は実測ではなく、公表来場者数にもとづく**推定**
- 追加スポット機能が使う全メッシュ表(mesh_levels.json)は2019年実測のままで、
  上記2箇所のような「2019年以降の新施設」は反映されない
- 時間帯カーブは全スポットともテンプレート(v0)。M8でv1化(出典裏付け)する
